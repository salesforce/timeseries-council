System Design

Architecture

A production-grade orchestration engine connecting LLMs, foundation models, and statistical methods into a unified analysis platform.

How It Works

1

Natural Language Input

Ask questions in plain English. The orchestrator parses intent and selects the right analysis tool.

2

Series Profiling

Compute 10 statistical features (mean, std, CV, autocorrelation, skewness, kurtosis) to characterize the data.

3

Model Selection

Cross-validation on held-out data computes MAE per model. An LLM reviews performance evidence and selects the optimal model or mixture.

4

Parallel Execution

Selected models run concurrently via ThreadPoolExecutor. Each model produces independent predictions.

5

Ensemble & Response

Results are aggregated via quantile-aware blending or majority voting, with optional context-aware adjustment. The LLM generates a natural-language explanation.

Multi-Agent Deliberation

The Council

Inspired by Andrej Karpathy's LLM Council architecture. Multiple AI experts deliberate through a structured 3-stage peer review process.

Mode 1

Standard

Single LLM analyzes the query, selects tools, and summarizes results. Fast and efficient for straightforward questions.

Mode 2

Simple Council

Three specialized roles analyze the same data from different perspectives: Quantitative, Risk, and Business.

Mode 3

Advanced Council

Karpathy-style 3-stage deliberation with 5 expert personas, anonymous peer review, and chairman synthesis.

Research Journey

Four Phases of Innovation

From literature review to production-grade system: a 16-month research journey.

Phase I
Aug - Oct 2024

Literature Review & Research Onboarding

  • Explored Salesforce Merlion, Moirai, GIFT-Eval, and TimeLLM
  • Surveyed 200+ slides of techniques across foundation models, causal forecasting, and probabilistic methods
  • Identified Moirai patch-size limitation as key research direction
  • Proposed Moirai-Ops zero-shot forecaster for operational data
Outcome: Problem formulation and research directions defined
Phase II
Nov 2024 - Jan 2025

Model Implementation & Preliminary Results

  • Implemented classical (ARIMA, SARIMA) and transformer-based models
  • Developed meta-learning prototype: 4-15% improvement over Moirai baseline
  • Rolling-based forecasting: 30% better MAE
  • Stochastic selection analysis: 41.6% potential improvement
  • Adapted GRPO (Group Relative Policy Optimization) from LLM training
Key Metrics: 4-15% meta-learning improvement, 30% MAE via rolling context
Phase III
Feb - Apr 2025

Advanced Frameworks & Research Paper

  • Tune-as-Inference: Amortized configuration learning using LightGBM learning-to-rank for per-series context length and patch size selection
  • 7.6% MAE improvement on Uber TLC, 8.0% on Electricity — within 3-5% of 20-trial Bayesian optimization at 20x lower cost
  • Peak Multiplier Method: 100% anomaly recall with 96.5% false-positive reduction
  • Validated on Moirai 1.1-R-small and PatchTST (model-agnostic)
  • Paper accepted at ICLR 2026 TSALM Workshop
Publication: "Tune-as-Inference: Amortized Configuration Learning for Time Series Foundation Models" — Accepted at ICLR 2026 TSALM Workshop
Phase IV
May 2025 - Present

Time Series Council: Research to Production

  • Synthesized all phases into ~19,264 lines of production Python
  • Tune-as-Inference insight evolved into LLM-driven model selection
  • Integrated 7 forecasters, 14 detectors, 5 LLM providers
  • Karpathy-style 3-stage council deliberation
Delivery: Production-grade Time Series Council system for AIOps

Tune-as-Inference: Core Research Contribution

Accepted — ICLR 2026 TSALM Workshop

Tune-as-Inference: Amortized Configuration Learning for Time Series Foundation Models — reformulates hyperparameter tuning as a single inference step using a lightweight learning-to-rank meta-learner.

Impact

Benefits to AIOps

How Time Series Council transforms operational intelligence at scale.

01

Intelligent Anomaly Detection

14 detector methods with LLM-driven per-metric selection eliminate manual tuning. Consensus voting with confidence scoring (high: 75%+, medium: 50%+) drastically reduces false positives.

02

Capacity Forecasting

7 forecasting models producing probabilistic outputs with confidence intervals. Quantile-aware ensemble blending and cross-validation-based selection improve accuracy by 21-28% over individual models.

03

Full Interpretability

Council mode provides quantitative, risk, and business perspectives. Every model selection decision comes with LLM-generated rationale. Per-model results preserved for drill-down.

04

Multi-Interface Operations

CLI for scripting, FastAPI web UI for interactive analysis, Python API for automation, and Jupyter notebooks for exploration. 5 LLM providers ensure no vendor lock-in.

05

Salesforce Ecosystem Integration

Incorporates Salesforce Merlion detectors and Moirai foundation models. Designed as baseline for Seamless Detection, tested on real CRM operational data.

06

Graceful Degradation

Optional dependencies degrade silently. Individual model failures don't crash the ensemble. LLM selection failures fall back to curated priority lists. Always produces a result.

Get Started

Quick Start

Installation 
# Install with all dependencies
pip install timeseries-council[all]

# Download model weights
timeseries-council download-models

# Check status
timeseries-council status
CLI Usage 
# Interactive chat with your data
timeseries-council chat data.csv \
  --target sales \
  --provider anthropic

# Launch web interface
timeseries-council serve \
  --host 0.0.0.0 --port 8000
Python API 
from timeseries_council import Orchestrator

orchestrator = Orchestrator(
    csv_path="data.csv",
    target_col="sales",
    provider="anthropic"
)

result = orchestrator.chat(
    "Will sales spike next week?"
)
print(result.response)
